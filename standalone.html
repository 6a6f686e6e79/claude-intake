claudeintake

Your Claude Memory Intake

Fill in as much or as little as you like, then click Generate to copy a memory bootstrap into claude.ai (or download .md files for Claude Code). Everything runs in your browser — nothing is uploaded.

🔒 This tool runs entirely in your browser. No data is sent to any server. Use Backup / Restore to keep a copy.
💾 Backup / Restore (download or load a JSON file with all your form data)

The local-storage save in your browser is convenient but tied to this one browser on this one device. A downloaded backup is the durable copy — keep it in your usual place (Dropbox / iCloud Drive / Google Drive / wherever).

The backup file is plain JSON you can read and edit. It includes a schemaVersion so future versions of this tool can migrate older backups forward.

Recover from claude.ai (incomplete — Backup / Restore above is the reliable path)

If you don't have a backup file, you can ask claude.ai to dump what it remembers. This path is incomplete by design: claude.ai stores some details as prose that don't always map cleanly to form fields, and Claude's wording varies between sessions. Prefer the Backup / Restore panel above when you have a saved file.

To attempt recovery from your current claude.ai memory:

  1. Click 📋 Copy prompt to clipboard below.
  2. Open a claude.ai chat and paste it. Send.
  3. Claude will reply with a fenced claude-intake-export block. Copy its entire reply.
  4. Paste that into the box below and click Load.

Already have a claude-ai-bootstrap.md file from a previous intake? Skip steps 1–3 and paste its contents directly.
